# Design Document — Transcriber

## Overview

Fitur Transcriber mengambil subtitle/caption dari video YouTube melalui endpoint `timedtext` YouTube, mem-parsing-nya menjadi array `Segment` terstruktur, dan menyajikannya di Web dengan opsi export TXT dan SRT.

Karena YouTube tidak menyediakan official API untuk transcript, fitur ini mengakses endpoint tidak resmi `https://www.youtube.com/api/timedtext` langsung dari Cloudflare Workers. Pendekatan ini menghindari kebutuhan API key untuk transcript dan bekerja selama endpoint tersebut tersedia.

Schema sudah ada di `packages/shared/src/index.ts` dan implementasi awal `transcript.ts` sudah ada. Design ini mendokumentasikan arsitektur lengkap, keputusan desain, dan correctness properties untuk memandu pengembangan dan testing.

## Architecture

```
┌─────────────────────────────────────────────────────────┐
│  Browser (SvelteKit)                                     │
│                                                          │
│  +page.svelte                                            │
│  ├── Form: input videoId/URL + language selector         │
│  ├── State: transcript, loading, error                   │
│  └── Export: formatTxt(), formatSrt() → download         │
└────────────────────┬────────────────────────────────────┘
                     │ GET /transcript?videoId=&lang=
                     ▼
┌─────────────────────────────────────────────────────────┐
│  Cloudflare Workers (Hono)                               │
│                                                          │
│  GET /transcript                                         │
│  ├── zValidator(TranscriptQuerySchema)                   │
│  ├── extractVideoId(input) — URL → ID                    │
│  ├── validateVideoId(id) — regex [a-zA-Z0-9_-]{11}       │
│  ├── fetchVideoTitle(videoId, apiKey) — non-fatal        │
│  └── fetchTranscript(videoId, lang)                      │
│      ├── attempt: lang, manual                           │
│      ├── attempt: en, manual (fallback)                  │
│      ├── attempt: lang, auto-generated                   │
│      └── attempt: en, auto-generated (fallback)          │
└────────────────────┬────────────────────────────────────┘
                     │ fetch timedtext JSON
                     ▼
┌─────────────────────────────────────────────────────────┐
│  YouTube timedtext endpoint                              │
│  https://www.youtube.com/api/timedtext                   │
│  ?v={videoId}&lang={lang}&fmt=json3[&kind=asr]           │
└─────────────────────────────────────────────────────────┘
```

**Keputusan desain:**

- Fallback chain (lang → en → lang+asr → en+asr) memaksimalkan kemungkinan mendapat transcript
- `videoTitle` diambil dari YouTube Data API secara non-fatal — gagal tidak memblokir response
- Rate limit 20 req/min per IP di level middleware, sebelum logika bisnis

## Components and Interfaces

### API Route: `GET /transcript`

```
GET /transcript?videoId={id|url}&lang={lang}
```

**Query params** (divalidasi via `TranscriptQuerySchema`):

- `videoId` — string, min 1 karakter. Bisa berupa 11-char ID atau full YouTube URL
- `lang` — string, default `"id"`

**Response sukses (200):**

```typescript
TranscriptResponse {
  videoId: string
  videoTitle?: string
  language: string        // bahasa yang berhasil diambil
  autoGenerated: boolean
  segments: TranscriptSegment[]
}
```

**Response error:**
| Status | Kondisi |
|--------|---------|
| 400 | videoId kosong/missing atau format tidak valid |
| 404 | Tidak ada transcript tersedia |
| 429 | Rate limit terlampaui |
| 502 | timedtext endpoint tidak dapat dijangkau |

### Parser: `parseTimedtext(data)`

Fungsi murni yang mengubah response JSON timedtext menjadi `TranscriptSegment[]`.

```typescript
type TimedtextEvent = {
  tStartMs: number
  dDurationMs: number
  segs?: Array<{ utf8: string }>
}

function parseTimedtext(events: TimedtextEvent[]): TranscriptSegment[]
```

- Filter events yang tidak memiliki `segs`
- Gabungkan semua `utf8` dalam satu event menjadi satu string
- Strip HTML entities (`&amp;`, `&lt;`, `&#39;`, dll) dan HTML tags
- Konversi `tStartMs / 1000` → `start` (detik), `dDurationMs / 1000` → `duration` (detik)
- Filter segment dengan text kosong setelah stripping

### URL Extractor: `extractVideoId(input)`

Fungsi murni yang mengekstrak 11-char video ID dari berbagai format input.

```typescript
function extractVideoId(input: string): string | null
```

Format yang didukung:

- `dQw4w9WgXcQ` — ID langsung
- `https://www.youtube.com/watch?v=dQw4w9WgXcQ`
- `https://youtu.be/dQw4w9WgXcQ`
- `https://www.youtube.com/embed/dQw4w9WgXcQ`
- `https://m.youtube.com/watch?v=dQw4w9WgXcQ`

### Export Formatters (Web)

Dua fungsi murni di `apps/web/src/lib/transcript.ts`:

```typescript
function formatTxt(segments: TranscriptSegment[]): string
function formatSrt(segments: TranscriptSegment[]): string
function formatSrtTimestamp(seconds: number): string // → "HH:MM:SS,mmm"
```

**TXT format:** setiap `segment.text` dipisahkan newline, tanpa timestamp.

**SRT format per segment:**

```
{index}
{HH:MM:SS,mmm} --> {HH:MM:SS,mmm}
{text}
{blank line}
```

### Shared Package: `packages/shared/src/index.ts`

Schema sudah ada (tidak perlu perubahan):

```typescript
TranscriptSegmentSchema // text, start, duration
TranscriptResponseSchema // videoId, videoTitle?, language, autoGenerated, segments
TranscriptQuerySchema // videoId (min 1), lang (default "id")
```

### Web Component: `apps/web/src/routes/+page.svelte`

State (Svelte 5 runes):

```typescript
let videoInput = $state('') // URL atau videoId dari user
let lang = $state('id') // bahasa yang dipilih
let transcript = $state<TranscriptResponse | null>(null)
let loading = $state(false)
let error = $state<string | null>(null)
```

## Data Models

### TranscriptSegment

```typescript
{
  text: string // teks bersih, tanpa HTML
  start: number // waktu mulai dalam detik (float)
  duration: number // durasi dalam detik (float)
}
```

### TranscriptResponse

```typescript
{
  videoId: string           // 11-char YouTube video ID
  videoTitle?: string       // judul video, opsional
  language: string          // kode bahasa yang berhasil diambil (e.g. "id", "en")
  autoGenerated: boolean    // true jika caption dari ASR YouTube
  segments: TranscriptSegment[]
}
```

### timedtext JSON3 format (upstream)

```typescript
{
  events: Array<{
    tStartMs: number // waktu mulai dalam milidetik
    dDurationMs: number // durasi dalam milidetik
    segs?: Array<{
      utf8: string // potongan teks
    }>
  }>
}
```

## Correctness Properties

_A property is a characteristic or behavior that should hold true across all valid executions of a system — essentially, a formal statement about what the system should do. Properties serve as the bridge between human-readable specifications and machine-verifiable correctness guarantees._

### Property 1: Response shape selalu lengkap

_For any_ valid `videoId` dan mock timedtext response yang valid, response API SHALL selalu mengandung field `videoId`, `language`, `autoGenerated`, dan `segments` (array).

**Validates: Requirements 1.2**

### Property 2: Lang parameter diteruskan ke timedtext

_For any_ kode bahasa yang valid, ketika `lang` diberikan dalam query, URL timedtext yang di-fetch SHALL mengandung parameter `lang` yang sama.

**Validates: Requirements 1.4**

### Property 3: Timedtext events → Segment fields

_For any_ array timedtext events yang valid (memiliki `segs`), setiap Segment yang dihasilkan parser SHALL memiliki `text` bertipe string, `start` bertipe number (dalam detik), dan `duration` bertipe number (dalam detik), dengan `start = tStartMs / 1000` dan `duration = dDurationMs / 1000`.

**Validates: Requirements 2.1**

### Property 4: HTML dibersihkan dari segment text

_For any_ timedtext segment yang mengandung HTML entities (`&amp;`, `&lt;`, `&gt;`, `&#39;`, `&quot;`) atau HTML tags (`<b>`, `<i>`, `<font>`, dll), teks yang dihasilkan parser SHALL tidak mengandung karakter HTML tersebut.

**Validates: Requirements 2.4**

### Property 5: Timedtext parse round-trip

_For any_ array `TranscriptSegment` yang valid, melakukan serialize ke JSON kemudian parse kembali SHALL menghasilkan array yang equivalent (sama panjang, setiap field identik).

**Validates: Requirements 2.5, 6.4**

_Catatan: Property 5 menggabungkan requirements 2.5 (parser round-trip) dan 6.4 (schema JSON round-trip) karena keduanya menguji invariant yang sama — serialisasi tidak boleh mengubah data._

### Property 6: Ekstraksi videoId dari berbagai format URL

_For any_ URL YouTube yang valid (youtube.com/watch?v=, youtu.be/, embed/, m.youtube.com/watch?v=), fungsi `extractVideoId` SHALL menghasilkan string 11 karakter yang match pattern `[a-zA-Z0-9_-]{11}`.

**Validates: Requirements 3.2**

### Property 7: videoId tidak valid selalu ditolak

_For any_ string yang tidak match pattern `[a-zA-Z0-9_-]{11}` (terlalu pendek, terlalu panjang, atau mengandung karakter tidak valid), API SHALL mengembalikan HTTP 400.

**Validates: Requirements 3.3**

### Property 8: Export format selalu benar

_For any_ array `TranscriptSegment` yang tidak kosong:

- `formatTxt(segments)` SHALL menghasilkan string di mana setiap `segment.text` muncul sebagai baris terpisah tanpa timestamp
- `formatSrt(segments)` SHALL menghasilkan string di mana setiap segment memiliki index, timestamp range format `HH:MM:SS,mmm --> HH:MM:SS,mmm`, teks, dan baris kosong sebagai separator

**Validates: Requirements 5.3, 5.4**

### Property 9: Nama file export konsisten

_For any_ `videoId` yang valid, nama file yang dihasilkan untuk download SHALL selalu `transcript-{videoId}.txt` untuk TXT dan `transcript-{videoId}.srt` untuk SRT.

**Validates: Requirements 5.5, 5.6**

## Error Handling

| Kondisi                       | HTTP Status | Pesan                                           |
| ----------------------------- | ----------- | ----------------------------------------------- |
| `videoId` missing/empty       | 400         | Pesan validasi Zod                              |
| `videoId` format tidak valid  | 400         | `"Invalid videoId format"`                      |
| timedtext endpoint non-200    | 502         | `"Failed to fetch transcript from YouTube"`     |
| Semua fallback attempts gagal | 404         | `"Transcript not available for this video."`    |
| Rate limit terlampaui         | 429         | `"Rate limit exceeded. Try again in a minute."` |

**Prinsip error handling:**

- `videoTitle` fetch gagal → swallow, lanjut tanpa title (non-fatal)
- Fallback chain: jika lang yang diminta tidak ada, coba `en`, lalu auto-generated
- Jangan expose internal error details ke client

## Testing Strategy

Sesuai TDD workflow project ini: Schema → Test → Impl → Green.

### Unit Tests (example-based)

**`packages/shared/src/index.test.ts`** — sudah ada, mencakup:

- `TranscriptQuerySchema`: default lang, reject empty videoId
- `TranscriptSegmentSchema`: parse valid segment
- `TranscriptResponseSchema`: parse valid response, optional videoTitle

**`apps/api/src/routes/transcript.test.ts`** — sudah ada, mencakup:

- 400 pada missing videoId
- 200 dengan segments yang benar (mock fetch)
- 404 ketika transcript tidak tersedia

**`apps/web/src/lib/transcript.test.ts`** — perlu dibuat:

- `extractVideoId`: berbagai format URL
- `formatTxt`: output tanpa timestamp
- `formatSrt`: format SRT standar, timestamp conversion

### Property-Based Tests

Library: **fast-check** (TypeScript, cocok untuk Vitest).

Setiap property test dikonfigurasi minimum 100 iterasi.

**`packages/shared/src/index.test.ts`** — tambahkan:

- Property 5: TranscriptResponse JSON round-trip

**`apps/api/src/routes/transcript.test.ts`** — tambahkan:

- Property 3: Timedtext events → Segment fields (generate berbagai event arrays)
- Property 4: HTML stripping (generate text dengan berbagai HTML entities/tags)
- Property 7: Invalid videoId → 400 (generate string yang tidak match pattern)

**`apps/web/src/lib/transcript.test.ts`** — tambahkan:

- Property 6: extractVideoId dari berbagai URL format
- Property 8: formatTxt dan formatSrt output format
- Property 9: Nama file export

Tag format untuk setiap property test:

```typescript
// Feature: transcriber, Property {N}: {property_text}
```

### Tidak Perlu Ditest

- UI rendering (Svelte components) — sesuai aturan project
- Behavior YouTube timedtext endpoint itu sendiri — external service
- `videoTitle` fetch — non-fatal, sudah di-swallow
