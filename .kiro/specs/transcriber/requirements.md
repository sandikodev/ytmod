# Requirements Document

## Introduction

Fitur Transcriber memungkinkan user mengambil transcript/subtitle dari video YouTube dan mengekspornya dalam format teks atau SRT. YouTube tidak menyediakan official API untuk transcript, sehingga fitur ini menggunakan endpoint `timedtext` YouTube secara langsung dari Cloudflare Workers. Fitur ini melengkapi Comment Downloader yang sudah ada dengan memberikan akses ke konten teks video.

## Glossary

- **Transcriber**: Komponen sistem yang mengambil dan memproses transcript dari YouTube
- **Transcript**: Teks subtitle/caption dari video YouTube beserta informasi waktu
- **Segment**: Satu unit transcript yang berisi teks, waktu mulai, dan durasi
- **timedtext**: Endpoint YouTube untuk mengambil data caption/subtitle
- **SRT**: Format file subtitle standar (SubRip Text) dengan timestamp
- **Auto-generated**: Caption yang dibuat otomatis oleh YouTube (bukan manual)
- **VideoId**: Identifier unik video YouTube (contoh: `dQw4w9WgXcQ`)
- **API**: Hono application yang berjalan di Cloudflare Workers
- **Web**: SvelteKit application yang berjalan di GitHub Pages
- **Parser**: Komponen yang mengubah response timedtext XML/JSON menjadi array Segment

## Requirements

### Requirement 1: Pengambilan Transcript dari YouTube

**User Story:** As a user, I want to fetch the transcript of a YouTube video by providing its URL or video ID, so that I can read the video content as text.

#### Acceptance Criteria

1. WHEN a valid `videoId` is provided, THE API SHALL fetch transcript data from the YouTube timedtext endpoint
2. WHEN a valid `videoId` is provided, THE API SHALL return a response containing `videoId`, `language`, `autoGenerated`, and `segments` array
3. WHEN the `lang` query parameter is omitted, THE API SHALL default to fetching the transcript in language `id`
4. WHEN a `lang` query parameter is provided, THE API SHALL attempt to fetch the transcript in the specified language
5. IF the YouTube timedtext endpoint returns a non-200 status, THEN THE API SHALL return an error response with HTTP status 502 and a descriptive message
6. IF no transcript is available for the requested video and language, THEN THE API SHALL return an error response with HTTP status 404 and message `"Transcript not available for this video"`

### Requirement 2: Parsing Transcript Segments

**User Story:** As a developer, I want the transcript data to be parsed into structured segments, so that the Web can display and export it correctly.

#### Acceptance Criteria

1. THE Parser SHALL convert each timedtext entry into a Segment object with `text` (string), `start` (number, seconds), and `duration` (number, seconds) fields
2. WHEN the timedtext response contains auto-generated captions, THE API SHALL set `autoGenerated` to `true` in the response
3. WHEN the timedtext response contains manually uploaded captions, THE API SHALL set `autoGenerated` to `false` in the response
4. THE Parser SHALL strip HTML entities and tags from segment text before returning
5. FOR ALL valid timedtext responses, parsing then re-serializing then parsing SHALL produce an equivalent array of Segments (round-trip property)

### Requirement 3: Validasi Input

**User Story:** As a user, I want clear error messages when I provide invalid input, so that I know how to correct my request.

#### Acceptance Criteria

1. IF the `videoId` query parameter is missing or empty, THEN THE API SHALL return HTTP status 400 with message `"videoId is required"`
2. WHEN a full YouTube URL is provided as `videoId`, THE API SHALL extract the video ID from the URL before processing
3. IF the extracted or provided `videoId` does not match the pattern `[a-zA-Z0-9_-]{11}`, THEN THE API SHALL return HTTP status 400 with message `"Invalid videoId format"`

### Requirement 4: Tampilan Transcript di Web

**User Story:** As a user, I want to see the transcript displayed as readable text in the browser, so that I can read the video content without watching it.

#### Acceptance Criteria

1. THE Web SHALL provide a form with an input field for YouTube video URL or video ID
2. THE Web SHALL provide a language selector with at least `id` (Indonesia) and `en` (English) as options
3. WHEN the user submits the form with a valid video ID, THE Web SHALL display a loading indicator while fetching
4. WHEN the API returns a successful response, THE Web SHALL display the transcript as flowing paragraphs with each segment on its own line
5. WHEN `autoGenerated` is `true` in the response, THE Web SHALL display a notice indicating the transcript is auto-generated
6. IF the API returns an error, THE Web SHALL display the error message to the user

### Requirement 5: Export Transcript

**User Story:** As a user, I want to export the transcript as a text file or SRT subtitle file, so that I can use it in other applications.

#### Acceptance Criteria

1. WHEN a transcript is loaded, THE Web SHALL display an "Export TXT" button that downloads the transcript as a `.txt` file
2. WHEN a transcript is loaded, THE Web SHALL display an "Export SRT" button that downloads the transcript as a `.srt` file
3. WHEN exporting as TXT, THE Web SHALL write each segment's `text` on a new line without timestamps
4. WHEN exporting as SRT, THE Web SHALL format each segment according to the SRT standard: index, timestamp range (`HH:MM:SS,mmm --> HH:MM:SS,mmm`), text, and blank line separator
5. THE Web SHALL name the downloaded TXT file `transcript-{videoId}.txt`
6. THE Web SHALL name the downloaded SRT file `transcript-{videoId}.srt`

### Requirement 6: Schema Shared

**User Story:** As a developer, I want shared Zod schemas for transcript data, so that the API and Web use consistent types without duplication.

#### Acceptance Criteria

1. THE Shared package SHALL export `TranscriptSegmentSchema` with fields `text` (string), `start` (number), `duration` (number)
2. THE Shared package SHALL export `TranscriptResponseSchema` with fields `videoId` (string), `videoTitle` (string, optional), `language` (string), `autoGenerated` (boolean), `segments` (array of TranscriptSegment)
3. THE Shared package SHALL export `TranscriptQuerySchema` with fields `videoId` (string, min length 1) and `lang` (string, default `"id"`)
4. FOR ALL valid `TranscriptResponseSchema` objects, serializing to JSON then parsing back SHALL produce an equivalent object (round-trip property)
